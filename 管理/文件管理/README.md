## 这套体系解决什么问题

很多人不是不会做事，而是桌面、项目、笔记和素材流转太乱，导致每天都在重复找东西、重复整理、重复丢失上下文。

这套体系的目标是：

- 减轻规划、执行、复盘三件事的阻力
- 把“想写的”“正在做的”“暂时没用的”分开
- 让 AI 可以辅助整理，但不让文件失控
- 用 Git 做备份，降低误删和丢失风险

## 一句话原则

**想写的放知识库，做的放 AIGC / 自媒体，杂物放仓库。**

## 结构总览

```text
桌面/
├── 马哲/        ← 指导思想
├── 知识库/      ← 日记、笔记
├── AIGC/        ← 代码项目
├── 自媒体/      ← 视频相关
└── 仓库/        ← 杂物
```

## 每天怎么用

| 时间 | 动作 | 结果 |
|:-----|:-----|:-----|
| 早上 | 写今日计划 | 最多三件事，降低分心 |
| 白天 | 做项目 / 学习 / 产出内容 | 文件进入正确目录 |
| 晚上 | 写复盘 | 把经验沉淀回知识库 |
| 周末 | 做周总结 | 汇总本周节奏与问题 |

常用触发词：

| 触发词 | 功能 |
|:-------|:-----|
| 早上好 | 写今日计划 |
| 你好 | AI 辅助整理文件 |
| 晚安 | 写每日复盘 |
| 周总结 | 写周复盘 |

## 人机协作

- AI 给出整理方案、命名建议、归档建议
- 人负责做最终决策，避免误移动、误删除
- Git 负责做历史记录与远程备份，保证文件安全

## 快速开始

1. 安装 `Git`
2. 注册 `GitHub`
3. 创建一个用于备份的仓库
4. 把自己的 `GitHub 用户名 / 邮箱 / 仓库地址` 告诉 AI

AI 接下来可以继续帮你：

- 初始化本地仓库
- 配置用户信息
- 关联远程仓库
- 建立基础目录结构

如果你要从零开始，先看 [docs/快速开始.md](docs/快速开始.md)。

## 各区域职责

| 区域 | 放什么 | 不放什么 |
|:-----|:-------|:---------|
| 马哲 | 指导思想、方法论 | 具体技术笔记 |
| 知识库 | 日记、笔记、复盘、想法 | 代码、视频素材 |
| AIGC | 代码项目、项目想法、开发过程 | 日记、复盘 |
| 自媒体 | 素材、草稿、成品 | 代码仓库 |
| 仓库 | 安装包、过时项目、杂物 | 正在用的核心资料 |

## 闭环

这套体系的关键不是“把文件分类好看”，而是让工作形成闭环：

```text
知识库（规划）
  ↓
AIGC / 自媒体（执行）
  ↓
知识库（总结与复盘）
```

也就是说：

- 知识库负责起点和终点
- AIGC / 自媒体负责中间的生产过程
- 仓库只是杂物箱，不参与闭环
