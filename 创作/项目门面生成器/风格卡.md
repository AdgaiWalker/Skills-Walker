# 风格卡

这份文档不是流程说明，也不是字段表。

它要解决的是另一个问题：

**同一个工作流里，不同产物应该“像什么”，不应该“像什么”。**

一句话理解：

`风格卡负责统一气质。`

---

## 为什么要有风格卡

没有风格卡时，最常见的问题是：

- README 写得像作者自言自语
- X 主帖写得像官号说明书
- 信息卡只是排字，不像编辑部图文
- SVG 标题只有形式，没有项目气质

这些问题不是字段缺失，也不是流程没走完，
而是：**产物气质不对。**

所以风格卡要做的，不是规定句子模板，而是规定：

- 应该更像什么
- 不应该像什么
- 什么是对味
- 什么是一眼就偏了

---

## 使用方式

风格卡不单独驱动一个产物。

它主要在这三个阶段起作用：

1. 上游生成时  
   约束语气、结构倾向和表达边界

2. 评测时  
   判断“气质对不对”，而不只看结构对不对

3. 修正时  
   告诉 AI 应该往哪一类感觉收，而不是盲目重写

---

## 风格卡 1：项目门面文风

### 它应该像什么

- 像一个可以直接给陌生人看的项目首页
- 像一个能快速判断“这是不是给我用的”的入口页
- 像一个已经整理好的门面，而不是思考草稿

### 它不应该像什么

- 不像作者的自言自语
- 不像纯提示词说明书
- 不像产品官宣腔
- 不像空洞品牌文案
- 不像为了“显得有深度”而故意绕

### 语气要求

- 直给
- 清楚
- 面向陌生人
- 能落地
- 少形容词，多判断

### 第一屏要求

第一屏优先解决：

- 这是什么
- 对谁有用
- 现在怎么开始
- 用完能到什么程度

第一屏不要优先塞：

- 作者的大段理念
- 设计过程回顾
- 过长的背景历史
- 过度的“愿景式表达”

### 常见对味信号

- 看完前两屏，读者已经知道这是不是给自己用的
- 能快速找到安装、入口、示例
- 就算不深读，也知道项目的大致价值

### 常见跑偏信号

- 看了很久还不知道这是什么
- 满篇都像“我为什么要做它”
- 结构很完整，但不像一个真正能交付的首页

---

## 风格卡 2：秋知发帖风格

### 它应该像什么

- 像创作者本人在说话
- 像在分享一个刚做出来、想发出去的东西
- 像有判断的人，而不是有任务的品牌号

### 它不应该像什么

- 不像官宣
- 不像功能堆砌
- 不像 README 摘抄
- 不像营销腔
- 不像空泛感慨

### 语气要求

- 真实
- 简洁
- 有判断
- 说人话
- 有一点温度，但不煽情

### 结构倾向

优先判断这条帖更适合：

- 结果展示
- 动机表达
- 前后对比

不是所有内容都先讲“为什么做”。  
冷启动时，有明确结果的，优先先把结果交代出来。

### 常见对味信号

- 前两行能停住人
- 看起来像人写的，不像系统输出
- 读完知道你在发什么
- 读者会想点开跟帖或链接

### 常见跑偏信号

- 把动机写成自我感动
- 把结果写成理念废话
- 什么都想说，主帖太满
- 图和文案重复同一层信息

---

## 风格卡 3：信息卡文风

### 它应该像什么

- 像编辑部做出来的信息卡
- 像有主次、有判断、有结构的图文页面
- 像“读一眼就知道先看哪”的视觉页面

### 它不应该像什么

- 不像长文截图
- 不像 PPT 拼块
- 不像把句子塞进盒子
- 不像平均分配空间的模板图

### 结构要求

- 必须有主标题
- 必须有主判断
- 必须有至少一个重模块
- 不能所有模块等权
- 不能只追求信息多，不追求阅读路径

### 常见对味信号

- 主标题先被看见
- 停 2 秒就知道核心判断
- 即使不细读，也能理解主要结论

### 常见跑偏信号

- 下半区空
- 四块平均
- 标题和正文抢位置
- 信息很多，但没有阅读节奏

---

## 风格卡 4：SVG 视觉门面

### 它应该像什么

- 像一个值得看一眼的视觉锚点
- 像这个项目自己的标题门面
- 静态成立，动态更强

### 它不应该像什么

- 不像普通文字截图
- 不像为了动而动
- 不像独立存在但跟项目没关系的炫技动画

### 结构要求

- 标题足够短
- 结构适合拆件
- 动画前后都成立
- 和项目定位一致

### 常见对味信号

- 静态看就有门面感
- 动画后更像“发布视觉”，不是“技术演示”
- 和 README、信息卡、发帖气质一致

### 常见跑偏信号

- 只有技巧，没有项目气质
- 标题太长，拆开后没有视觉重心
- 动态成立，静态不成立

---

## 风格卡之间的关系

- `项目门面文风` 负责 README
- `秋知发帖风格` 负责 X 等发帖内容
- `信息卡文风` 负责图文卡
- `SVG 视觉门面` 负责标题、横幅和动画门面

它们不是四套分裂风格，而是同一条链上的不同外观：

- README 负责让人看懂
- 发帖负责让人停下
- 信息卡负责让人一眼抓住
- SVG 门面负责让人记住

---

## 一句话结论

风格卡的作用，不是让所有产物看起来一样，
而是：

**让它们在不同形式下，仍然像同一个项目发出来的东西。**
