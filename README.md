秋知的自创自用 AI 技能库，按“我要做什么”组织，而不是按零散 Prompt 或技术栈堆目录。  
它的目标是把可复用的工作方法沉淀成 skill，让 AI 在有上下文时更像会干活的搭档。

## 快速使用

1. 先让 AI 读取你的项目、文档、目录或已有内容。
2. 选择对应分类下的 `skill.md`。
3. 把 skill 交给 AI，并直接说你的任务。
4. AI 负责分析与执行，你只负责做必要决策。

## 这个仓库的原则

- 人做决策，AI 做执行
- 先吃上下文，再产出结果
- skill 要能直接用，不做提示词展览
- 按动作意图组织，而不是按工具名硬分类

## 仓库结构

| 分类 | 心智 | 适合内容 |
|:-----|:-----|:---------|
| OpenClaw | AI 代理专项 | 安装、配置、网关 |
| 管理 | 我要理一理 | 决策、文件、规划、配置 |
| 创作 | 我要写 / 做东西 | 文档、脚本、内容 |
| 元能力 | 我要变强 | 技能生成、方法沉淀 |

## 推荐使用方式

如果你是第一次使用这个仓库，建议优先从这两个方向开始：

- 需要把项目文档快速做成品：用 [README 生成器](创作/README%20生成器/)
- 需要把一段高质量对话沉淀成长期可复用能力：用 [技能生成](元能力/技能生成/)
